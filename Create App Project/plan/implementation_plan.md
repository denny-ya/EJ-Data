```powershell

```

**대처**: PowerShell을 관리자 권한으로 실행

## 🔗 Google Sheets API 연동 방법

### 옵션 A: Google Apps Script를 API로 활용 (🔴 추천)

**중요도**: 🔴 높음 | **우선순위**: 1순위 | **난이도**: ⭐ 쉬움

현재 Apps Script를 그대로 백엔드로 사용하고, 앱에서 호출

```javascript
// Apps Script에서 doGet/doPost 함수로 API 엔드포인트 제공
function doGet(e) {
  const action = e.parameter.action;
  if (action === 'search') {
    return ContentService.createTextOutput(JSON.stringify(searchData(e.parameter.query)));
  }
}
```

```typescript
// React Native에서 호출
const response = await fetch('YOUR_APPS_SCRIPT_URL?action=search&query=검색어');
const data = await response.json();
```

#### 📌 리스크 및 대처 방법

| 리스크 | 발생 가능성 | 영향도 | 대처 방법 |
|--------|-------------|--------|-----------|
| Apps Script 응답 지연 | 🟡 중간 | 🟡 중간 | 로딩 UI 표시, 타임아웃 설정 |
| CORS 에러 발생 | 🟢 낮음 | 🔴 높음 | Apps Script는 기본 CORS 허용 |
| API URL 노출 | 🟡 중간 | 🟡 중간 | URL 환경변수로 관리 |

```powershell
# 프로젝트에서 EAS 초기화
eas build:configure

# Android APK 빌드 (클라우드)
eas build --platform android --profile preview
```

### 빌드 프로필 설정 (`eas.json`)
```json
{
  "build": {
    "preview": {
      "android": {
        "buildType": "apk"
      }
    },
    "production": {
      "android": {
        "buildType": "app-bundle"
      }
    }
  }
}
```

 ## 📱 모바일 앱 필수 표준 및 기능 추가
 
 > **⚠️ 중요**: 단순 웹앱이 아닌 "네이티브 앱"으로서 완성도를 높이기 위해 다음 항목들이 반드시 포함되어야 합니다.
 
 ### 1. 🛡️ 모니터링 및 분석 (필수)
 앱이 죽거나 에러가 났을 때 개발자가 알 수 있어야 합니다.
 
 | 기능 | 추천 도구 | 설명 | 비용 |
 |------|-----------|------|------|
 | **Crash Reporting** | **Firebase Crashlytics** (추천) 또는 Sentry | 앱 충돌 시 로그 자동 수집 | 무료 |
 | **Analytics** | **Firebase Analytics** | 사용자 수, 버튼 클릭 등 행동 분석 | 무료 |
 
 ### 2. 🔄 OTA 업데이트 (Expo Updates)
 스토어 심사 없이 JS 코드만 즉시 수정하여 배포할 수 있는 강력한 기능입니다.
 
 - **활용**: 긴급 버그 수정 시 사용
 - **주의**: 네이티브 모듈(카메라, 권한 등) 변경 시에는 스토어 심사 필요
 
 ### 3. 🔗 딥링크 (Deep Linking)
 외부(문자, 카톡)에서 링크를 누르면 앱이 열리고 특정 화면으로 이동하는 기능입니다.
 - **설정**: `app.json`에 `scheme` 설정 (예: `hym-project://`)
 
 ### 4. 🔐 보안 및 권한 (Permissions)
 - **권한 요청**: 카메라, 알림, 위치 등 사용 시 **런타임 권한 요청** 필수
 - **보안 저장소**: 토큰/비밀번호는 `AsyncStorage`가 아닌 `Expo SecureStore` 사용
 
 ---
 
 ### 5. 📱 고급 하드웨어 기능 (Advanced Hardware Features)
 
 > **분석 배경**: `Beginner Tutorial` 비교 결과, 필수 하드웨어 기능이 누락되어 추가함.
 > **참고**: 초기 앱은 데이터 관리에 집중하므로, 아래 하드웨어 기능은 **낮은 우선순위 (Low)**로 배정합니다.
 
 | 기능 | 라이브러리 | 용도 | 중요도 |
 |------|------------|------|--------|
 | **카메라/갤러리** | `expo-image-picker` | 수리 전/후 사진 촬영 및 업로드 | 🟢 낮음 |
 | **위치 정보** | `expo-location` | 수리 현장 위치 자동 기록 | 🟢 낮음 |
 | **진동/햅틱** | `expo-haptics` | 버튼 터치, 성공/실패 피드백 | 🟢 낮음 |
 | **알림** | `expo-notifications` | 수리 일정 임박 알림 (로컬) | 🟢 낮음 |
