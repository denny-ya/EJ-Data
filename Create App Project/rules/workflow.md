# 🔄 작업 흐름 (Workflow)

> 앱 개발의 전체 프로세스와 각 단계별 규칙을 정의합니다.

---

## 📊 전체 워크플로우

```
┌─────────────────────────────────────────────────────────────────────────┐
│                                                                         │
│   1. 계획    →    2. 설계    →    3. 개발    →    4. 테스트    →    5. 배포   │
│  (Planning)     (Design)      (Development)    (Testing)      (Deploy)  │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

---

## 1️⃣ 계획 (Planning)

### 목표
요구사항을 정의하고 개발 범위를 확정합니다.

### 산출물
- [ ] 기능 요구사항 목록
- [ ] 우선순위 정의
- [ ] 일정 계획

### 진행 규칙

| 규칙 | 설명 |
|------|------|
| 요구사항 문서화 | 모든 요구사항은 문서로 정리 |
| 우선순위 결정 | 높음/중간/낮음으로 분류 |
| 범위 확정 | MVP(최소 기능)를 먼저 정의 |
| 일정 산정 | 각 기능별 예상 소요 시간 산정 |

### 체크리스트
```
[ ] 사용자 스토리 작성 완료
[ ] 기능 목록 확정
[ ] 우선순위 결정
[ ] 일정 계획 수립
[ ] 팀 역할 분담 (해당 시)
```

---

## 2️⃣ 설계 (Design)

### 목표
UI/UX를 설계하고 기술 아키텍처를 확정합니다.

### 산출물
- [ ] 화면 설계서 (와이어프레임/목업)
- [ ] 데이터 구조 정의
- [ ] API 설계 문서
- [ ] 설계 검토 체크리스트 완료

### 진행 규칙

| 규칙 | 설명 |
|------|------|
| 설계 우선 | 코딩 전 반드시 설계 완료 |
| 검토 필수 | `design-review-checklist.md` 완료 필수 |
| 승인 후 진행 | 설계 승인 후 개발 착수 |
| 변경 관리 | 설계 변경 시 문서 업데이트 필수 |

### 의사결정 프로세스

```
설계 초안 작성
       ↓
  팀 검토 회의
       ↓
   피드백 반영
       ↓
  최종 승인 ←─ 미승인 시 재검토
       ↓
   개발 착수
```

### 체크리스트
```
[ ] 전체 화면 목록 정의
[ ] 각 화면별 레이아웃 설계
[ ] 네비게이션 플로우 정의
[ ] 데이터 스키마 정의
[ ] API 엔드포인트 정의
[ ] 설계 검토 완료
```

---

## 3️⃣ 개발 (Development)

### 목표
설계에 따라 기능을 구현합니다.

### 산출물
- [ ] 소스 코드
- [ ] 코드 리뷰 완료

### 진행 규칙

| 규칙 | 설명 |
|------|------|
| 브랜치 생성 | 기능별 브랜치에서 개발 |
| 일일 커밋 | 최소 1일 1커밋 |
| 코드 리뷰 | PR 생성 후 리뷰 필수 |
| 테스트 작성 | 주요 기능은 테스트 코드 작성 |

### 개발 순서

```
1. 기본 레이아웃/네비게이션 구현
       ↓
2. 정적 UI 구현 (더미 데이터)
       ↓
3. API 연동
       ↓
4. 상태 관리 구현
       ↓
5. 에러 처리/예외 케이스
       ↓
6. UI 다듬기/애니메이션
```

### 체크리스트
```
[ ] 브랜치 생성
[ ] 기능 구현
[ ] 자체 테스트
[ ] 코드 정리 (lint, format)
[ ] PR 생성
[ ] 코드 리뷰 받기
[ ] 리뷰 반영 및 머지
```

---

## 4️⃣ 테스트 (Testing)

### 목표
버그를 발견하고 품질을 확보합니다.

### 산출물
- [ ] 테스트 결과 리포트
- [ ] 버그 목록 및 수정 내역

### 진행 규칙

| 규칙 | 설명 |
|------|------|
| 기능 테스트 | 모든 기능 수동 테스트 |
| 다바이스 테스트 | 최소 2개 이상 기기에서 테스트 |
| 에지 케이스 | 예외 상황 테스트 |
| 버그 기록 | 발견된 버그는 모두 기록 |

### 테스트 항목

| 카테고리 | 테스트 항목 |
|----------|-------------|
| 기능 | 각 기능이 정상 동작하는가? |
| UI | 화면이 설계대로 표시되는가? |
| 네트워크 | 오프라인/느린 연결에서 동작하는가? |
| 입력 | 잘못된 입력에 대해 적절히 처리하는가? |
| 성능 | 로딩 시간이 적절한가? |

### 체크리스트
```
[ ] 전체 기능 테스트 완료
[ ] 다양한 기기 테스트
[ ] 엣지 케이스 테스트
[ ] 버그 수정 완료
[ ] 재테스트 완료
```

---

## 5️⃣ 배포 (Deploy)

### 목표
앱을 빌드하고 사용자에게 배포합니다.

### 산출물
- [ ] APK/AAB 파일
- [ ] Play Store 등록 (선택)

### 진행 규칙

| 규칙 | 설명 |
|------|------|
| 버전 관리 | 배포 전 버전 번호 업데이트 |
| 빌드 테스트 | 릴리스 빌드 테스트 필수 |
| 변경 로그 | 변경 사항 기록 |
| 백업 | 이전 버전 백업 보관 |

### 배포 순서

```
1. 코드 프리즈 (개발 중단)
       ↓
2. 최종 테스트
       ↓
3. 버전 번호 업데이트
       ↓
4. 릴리스 빌드
       ↓
5. 빌드 파일 테스트
       ↓
6. 배포/업로드
       ↓
7. 배포 확인
```

### 체크리스트
```
[ ] 버전 번호 업데이트
[ ] 변경 로그 작성
[ ] 릴리스 빌드 생성
[ ] 빌드 테스트
[ ] 스토어 업로드 (해당 시)
[ ] 배포 완료 확인
```
