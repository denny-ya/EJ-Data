### 3.2 API 엔드포인트 정의

| 기능 | Method | Endpoint | Request | Response |
|------|--------|----------|---------|----------|
| 전체 조회 | GET | `/getAll` | - | `{ items: [...] }` |
| 검색 | GET | `/search?q=` | query | `{ results: [...] }` |
| 상세 조회 | GET | `/getById?id=` | id | `{ item: {...} }` |
| 생성 | POST | `/create` | body | `{ success: true }` |
| 수정 | PUT | `/update` | body | `{ success: true }` |
| 삭제 | DELETE | `/delete?id=` | id | `{ success: true }` |

### 3.3 데이터 스키마 정의

```typescript
  id: string;
  title: string;
  description?: string;
  category?: string;
  createdAt: string;
  updatedAt: string;
}
```

### 7.4 리스크 평가 기준

**발생 가능성**
| 등급 | 설명 |
|------|------|
| 🔴 높음 | 70% 이상 발생 예상 |
| 🟡 중간 | 30~70% 발생 예상 |
| 🟢 낮음 | 30% 미만 발생 예상 |

**영향도**
| 등급 | 설명 |
|------|------|
| 🔴 높음 | 프로젝트 중단/실패 가능 |
| 🟡 중간 | 일정 지연/품질 저하 가능 |
| 🟢 낮음 | 경미한 영향, 쉽게 해결 가능 |

### 7.5 리스크 대응 우선순위

```
우선순위 = 발생 가능성 × 영향도

🔴🔴 = 최우선 대응 (즉시 대응 방안 마련)
🔴🟡 / 🟡🔴 = 높은 우선순위 (대응 방안 필수)
🟡🟡 = 중간 우선순위 (모니터링 및 대비)
🟢🟡 / 🟡🟢 = 낮은 우선순위 (인지만 하고 넘어감)
🟢🟢 = 최저 우선순위 (무시 가능)
```
